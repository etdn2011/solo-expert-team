# 精简版蒸馏协议 · Rapid Distillation Protocol

> 总管系统外部招募新专家时使用
> 目标：60分钟内完成可用Skill
> 质量目标：70分（可用）vs 完整版90分（优秀）

---

## 时间分配（60分钟）

| 阶段 | 时间 | 产出 |
|------|------|------|
| 候选人搜索 | 15分钟 | 3-5位候选人名单 |
| 用户确认 | 5分钟 | 选定1位候选人 |
| 信息收集 | 20分钟 | 核心观点素材 |
| Skill撰写 | 15分钟 | 完整SKILL.md |
| 质量检查 | 5分钟 | 通过/返工/放弃 |

---

## Phase 1: 候选人搜索（15分钟）

### 搜索策略

**关键词组合**：
```
[领域] + [细分话题] + expert/author/speaker
例如：
- "Gen Z mental health" + psychologist + author
- "SaaS pricing" + expert + book
- "climate tech investing" + VC + partner
```

**搜索渠道优先级**：
1. Google Scholar（学术论文）
2. TED.com（演讲者）
3. Amazon Books（畅销书作者）
4. LinkedIn（行业领袖）
5. 专业媒体（Harvard Business Review, MIT Tech Review等）

### 筛选标准（5分钟内判断）

| 标准 | 检查项 | 通过门槛 |
|------|--------|---------|
| **可蒸馏性** | 是否有公开著作/演讲/论文？ | 至少有3篇以上深度内容 |
| **权威性** | 在该细分领域是否被认可？ | 有权威背书（奖项、职位、引用） |
| **独特性** | 观点是否有独特性？ | 不是泛泛而谈，有具体方法论 |
| **时效性** | 观点是否过时？ | 近5年内有活跃输出 |

### 输出格式

```markdown
## 候选人名单

### 候选人1: [姓名]
- **身份**: [职位/头衔]
- **核心著作**: [书名/论文]
- **推荐理由**: [一句话]
- **可蒸馏性评分**: [1-10]
- **风险**: [信息不足/观点分散/时效性问题]

### 候选人2: ...
```

---

## Phase 2: 信息收集（20分钟）

### 核心信息源（优先级排序）

1. **著作/论文**（10分钟）
   - 阅读前言/结论/核心章节
   - 提取3-5个核心观点
   - 记录具体案例和证据

2. **演讲/访谈**（5分钟）
   - TED演讲、播客、访谈
   - 捕捉表达风格和标志性语句
   - 记录即兴回答中的真实观点

3. **社交媒体/博客**（3分钟）
   - Twitter/X、LinkedIn、个人博客
   - 了解最新观点和互动风格
   - 捕捉日常表达习惯

4. **他人评价**（2分钟）
   - 书评、采访、同行评价
   - 了解争议点和盲区
   - 确认诚实边界

### 信息提取模板

```markdown
## 原始素材

### 核心观点（从著作/论文提取）
1. [观点1] - 来源：[书名/论文名]
2. [观点2] - 来源：[书名/论文名]
3. [观点3] - 来源：[书名/论文名]

### 表达风格（从演讲/访谈提取）
- 常用句式：...
- 标志性词汇：...
- 语气特点：...
- 幽默风格：...

### 争议/局限（从评价提取）
- [争议1] - 来源：...
- [盲区1] - 来源：...
```

## Phase 3: Skill撰写（15分钟）

### 必须包含的Section（精简版）

#### 1. Frontmatter（1分钟）
```yaml
---
name: [name]-perspective-v2
description: |
  [一句话概括核心能力]
  【精简版蒸馏】基于[著作/演讲]快速提炼，Beta版本
---
```

#### 2. 角色扮演规则（1分钟）
- 直接以"我"回应
- 免责声明（首次激活时说一次）
- 退出角色指令

#### 3. 核心心智模型（8分钟）【重点】

**数量**：3-4个（完整版6个）

**每个模型包含**：
```markdown
### 模型X: [名称]

**一句话**：[核心洞察]

**证据**：[来源，一句话]

**应用**：[什么情况下使用]

**局限**：[什么情况下失效]
```

**快速提炼技巧**：
- 从著作目录/章节标题找模型线索
- 从演讲中的"我的三个观点"类结构提取
- 从反复出现的概念/框架确认

#### 4. 决策启发式（2分钟）

**数量**：5-6条（完整版8-10条）

**格式**：
```markdown
1. **[情境]** → [行动]
2. **[情境]** → [行动]
```

#### 5. 表达DNA（1分钟）

快速捕捉：
- **句式**：短句/长句/反问/排比？
- **词汇**：专业术语密度？标志性词汇？
- **确定性**："我认为" vs "这是事实"？
- **语气**：学术/亲和/犀利/温和？

#### 6. 诚实边界（1分钟）【必须】

至少3条：
```markdown
### 诚实边界

1. **信息时效性**：本Skill基于[时间]前的公开信息
2. **蒸馏局限性**：精简版可能遗漏 nuances
3. **领域边界**：我在[领域]有经验，在[领域]是盲区
```

#### 7. 调研信息（1分钟）
```markdown
---

## 调研信息

- **蒸馏时间**：[日期]
- **信息来源**：[主要来源列表]
- **版本**：精简版v1.0（Beta）
- **质量评分**：70/100（可用，但非最优）
```

---

## Phase 4: 质量检查（5分钟）

### 检查清单

| 检查项 | 标准 | 不通过处理 |
|--------|------|-----------|
| ☑️ 有3+核心模型 | 每个模型有4要素 | 补充WebSearch |
| ☑️ 有5+决策启发式 | 具体可操作 | 从素材中提取更多 |
| ☑️ 有诚实边界 | ≥3条 | 必须补充 |
| ☑️ 有身份一致性 | 表达DNA符合人物 | 调整用词 |
| ☑️ 无编造 | 所有观点有来源 | 删除或标注"推断" |

### 评分标准

| 分数 | 状态 | 处理 |
|------|------|------|
| 80-100 | 优秀 | 直接发布 |
| 60-79 | 可用 | 发布，标记Beta |
| 40-59 | 勉强 | 用户确认后发布 |
| <40 | 不可用 | 放弃，尝试下一个候选人 |

## 与完整版蒸馏的差异

| 维度 | 精简版（60分钟） | 完整版（2小时） |
|------|-----------------|----------------|
| **心智模型** | 3-4个 | 6-7个 |
| **决策启发式** | 5-6条 | 8-10条 |
| **表达DNA** | 基础版 | 详细版（句式/词汇/节奏/幽默/确定性） |
| **价值观** | 简略 | 完整（核心价值观+反模式+内在张力） |
| **智识谱系** | 无 | 完整（先驱/同代/后代/批评者） |
| **Agentic Protocol** | 简化 | 完整（双模式+强制WebSearch） |
| **质量目标** | 70分（可用） | 90分（优秀） |

## 使用建议

### 何时使用精简版？
- 问题紧急，需要快速响应
- 该领域现有专家覆盖度<50%
- 用户愿意接受Beta质量

### 何时升级到完整版？
- 精简版专家被频繁调用
- 用户反馈精简版质量不够
- 该领域成为高频需求

### 升级路径
```
精简版（60分钟，70分）
    ↓ 高频使用/用户反馈
完整版（额外60分钟，90分）
    ↓ 持续优化
专家Skill v2.0（持续迭代）
```

## 示例：精简版 vs 完整版

### 精简版（15分钟撰写）

```markdown
## 核心心智模型

### 模型1: 数字原住民心理
**一句话**：Z世代是数字原住民，他们的心理健康需求与数字体验密不可分。
**证据**：基于《iGen》研究，Z世代每天使用社交媒体3+小时
**应用**：设计APP时，社交功能不是附加，是核心
**局限**：不适用于非数字原住民群体

### 模型2: 焦虑的社交化
**一句话**：Z世代的焦虑很大程度上来自社交比较和FOMO（错失恐惧）
**证据**：Instagram使用与抑郁症状正相关（Twenge研究）
**应用**：APP设计要减少社交比较触发，增加真实连接
**局限**：焦虑也有非社交来源（学业、家庭等）

### 模型3: 即时满足 vs 长期福祉
**一句话**：Z世代习惯即时反馈，但心理健康需要长期投入
**证据**：TikTok的即时反馈机制 vs 心理治疗的延迟满足
**应用**：用游戏化设计 bridge 即时满足和长期目标
**局限**：过度游戏化可能削弱严肃性
```

### 完整版（额外60分钟补充）
- 增加3个模型（数字身份、算法影响、代际创伤）
- 每个模型补充更多证据和案例
- 增加详细表达DNA（Jean Twenge的学术严谨+媒体友好风格）
- 增加智识谱系（与Jonathan Haidt、Sherry Turkle的对话）
- 完善Agentic Protocol（强制WebSearch最新研究）
