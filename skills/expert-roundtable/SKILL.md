# 专家组圆桌辩论

> 「一个问题的答案，往往藏在多个视角的交汇处。」

## 核心理念

**不是找一个专家，而是让一群专家互相碰撞。**

传统用法：
- ❌ 「用乔布斯的视角分析这个产品」（单视角）
- ❌ 「找个专家看看」（智能路由，但仍是独立回答）

圆桌辩论用法：
- ✅ 「让专家们讨论一下这个产品策略」
- ✅ 「专家组圆桌分析这个投资决策」
- ✅ 「多角度解剖这个问题」
- ✅ 「辩证分析一下这个商业模式」

**圆桌辩论 vs 单视角分析的区别：**

| 维度 | 单视角分析 | 圆桌辩论 |
|------|-----------|---------|
| 视角数量 | 1个 | 2-4个 |
| 观点碰撞 | 无 | 有交叉讨论和交锋 |
| 盲区覆盖 | 依赖单个角色 | 角色之间互相补盲区 |
| 结论质量 | 单一视角深度 | 多角度综合判断 |
| 适用场景 | 明确领域问题 | 复杂、模糊、跨领域问题 |

## 圆桌辩论流程

### Step 1: 问题分析

收到用户问题后，先进行结构化分析：

| 分析维度 | 判断要点 |
|---------|---------|
| **核心议题** | 用户真正想解决什么问题？ |
| **涉及领域** | 产品/投资/营销/技术/心理/组织/人生... |
| **问题类型** | 决策型 / 分析型 / 创意型 / 危机型 |
| **复杂度** | 单一维度 / 多维度交叉 / 高度不确定 |
| **情绪背景** | 焦虑 / 迷茫 / 兴奋 / 困惑 / 紧迫 |

输出格式：

```
## 问题分析

- **核心议题**：[一句话概括]
- **涉及领域**：[列出相关领域]
- **问题类型**：[决策型/分析型/创意型/危机型]
- **复杂度评估**：[低/中/高]
```

### Step 2: 专家选择

根据问题分析结果，从已安装的专家skill中选择2-4位专家。

**选择原则：**
1. **覆盖性**：专家组合应覆盖问题的主要维度
2. **互补性**：专家之间应有不同甚至对立的视角
3. **冲突性**：至少安排一对可能产生观点碰撞的专家
4. **精准性**：优先选择与问题领域最匹配的专家

### Step 3: 逐位发言

每位专家从自己的心智模型出发，独立分析问题。

**关键要求：**
- **必须先读取对应专家的SKILL.md**，使用Read工具读取 `/workspace/.claude/skills/[专家skill目录名]/SKILL.md`
- 每位专家的回答必须符合其独特的心智模型和表达风格
- 专家之间此时不互动，各自独立分析
- 每位专家的发言应包含：核心观点、分析框架、具体建议

输出格式：

```
## 第一轮：独立发言

### 🎙️ [专家1中文名]（[英文名]）的视角

> *[一句该专家风格的标志性开场]*

**核心观点**：[一句话总结]

**分析**：
[从该专家的心智模型出发，对问题进行深度分析，2-4段]

**建议**：
[该专家会给出的具体行动建议，2-3条]

---

### 🎙️ [专家2中文名]（[英文名]）的视角

> *[一句该专家风格的标志性开场]*

...（同上格式）

---

### 🎙️ [专家3中文名]（[英文名]）的视角

> *[一句该专家风格的标志性开场]*

...（同上格式）
```

### Step 4: 交叉讨论

专家之间互相补充、质疑、反驳，至少进行1轮交锋。

**交锋规则：**
1. 每位专家至少回应一位其他专家的观点
2. 回应可以是：补充延伸、提出质疑、直接反驳、或指出盲区
3. 被质疑的专家可以进行回应
4. 鼓励发现观点之间的张力和矛盾

**交锋类型：**

| 类型 | 说明 | 示例 |
|------|------|------|
| **补充延伸** | 在另一位专家观点基础上进一步展开 | 「我同意X的观点，但还想补充一点...」 |
| **提出质疑** | 对另一位专家的假设或结论提出疑问 | 「X的框架忽略了一个关键变量...」 |
| **直接反驳** | 明确不同意另一位专家的结论 | 「我不同意X的判断，原因在于...」 |
| **指出盲区** | 指出另一位专家视角的局限性 | 「X的分析在Y场景下成立，但在Z场景下...」 |

输出格式：

```
## 第二轮：交叉讨论

### 💬 [专家A] 回应 [专家B]

[具体回应内容，1-2段]

### 💬 [专家B] 回应 [专家A]

[具体回应内容，1-2段]

### 💬 [专家C] 回应 [专家A] 和 [专家B]

[具体回应内容，1-2段]
```

### Step 5: 汇总总结

综合所有专家的观点和交锋，提炼出最终结论。

**总结结构：**

```
## 圆桌总结

### ✅ 共识点

[列出所有专家基本同意的观点，2-4条]

### ⚡ 分歧点

[列出专家之间存在明显分歧的地方，1-3条，说明各方立场]

### 🎯 综合建议

[基于共识和分歧，给出可执行的综合建议，3-5条]

### 📊 专家意见矩阵

| 观点/建议 | 专家A | 专家B | 专家C |
|-----------|-------|-------|-------|
| [观点1] | ✅ 支持 | ⚠️ 有保留 | ❌ 反对 |
| [观点2] | ✅ 支持 | ✅ 支持 | ✅ 支持 |
| [观点3] | ❌ 反对 | ✅ 支持 | ⚠️ 有保留 |
```

## 诚实边界

### 我能做到的

- **智能选择专家组合**：根据问题领域自动匹配最合适的2-4位专家
- **忠实还原专家视角**：通过读取每位专家的SKILL.md，确保分析符合其独特的心智模型
- **制造有意义的观点碰撞**：安排互补甚至对立的专家，产生真正的辩论价值
- **结构化输出**：提供清晰的共识、分歧和综合建议

### 我做不到的

- **保证专家观点的绝对准确性**：专家观点基于skill蒸馏，是对其思想体系的近似模拟，非本人真实观点
- **替代用户做最终决策**：圆桌讨论提供多角度分析，但决策权始终在用户手中
- **处理完全超出所有专家能力圈的问题**：如果问题涉及所有专家的盲区，会诚实说明
- **模拟专家之间的实时真实辩论**：交锋是基于各专家思想体系的逻辑推演，而非真实的对话互动

### 使用建议

- 对于**高度专业化**的问题（如医学诊断、法律建议），本系统不适用，请寻求专业人士帮助
- 对于**简单事实查询**，不需要启动圆桌辩论，直接回答即可
- 对于**价值观敏感**的问题，专家观点可能存在冲突，请结合自身判断
- 用户可以**指定参与讨论的专家**，例如：「让芒格和塔勒布讨论一下这个投资策略」

## 执行检查清单

每次执行圆桌辩论时，按以下检查清单确保质量：

- [ ] Step 1: 是否完成了问题分析（核心议题、涉及领域、问题类型、复杂度）？
- [ ] Step 2: 是否选择了2-4位专家？是否说明了选择理由？专家组合是否有互补性和冲突性？
- [ ] Step 3: 是否读取了每位专家的SKILL.md？每位专家的发言是否符合其独特风格？
- [ ] Step 4: 是否进行了至少1轮交叉讨论？交锋是否有实质性内容（非客套话）？
- [ ] Step 5: 是否提供了共识点、分歧点和综合建议？是否有专家意见矩阵？
