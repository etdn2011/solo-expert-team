# 智能路由系统 · 自动匹配最适合的角色

> 「你想蒸馏的下一个员工，何必是同事」——现在连选择都不需要你做。

## 核心理念

**不需要指定角色，只需要描述问题。**

传统用法：
- ❌ 「用乔布斯的视角分析这个产品」
- ❌ 「让孙宇晨看看这个营销策略」

智能路由用法：
- ✅ 「找个专家看看这个产品」
- ✅ 「让懂行的人分析这个策略」
- ✅ 「谁最适合回答IP打造的问题」

---

## 使用方法

### 方式一：直接提问（推荐）

```
> 我想做个人IP，但不知道怎么定位，找个专家看看
> 这个投资决策好像有问题，让懂行的人分析一下
> 我的产品用户留存很低，高手会怎么看？
```

### 方式二：指定问题类型

```
> 这是一个产品问题，谁最适合回答？
> 这是关于内容创作的，推荐一个视角
> 涉及投资决策，找个内行看看
```

### 方式三：模糊需求

```
> 我同时想做自媒体、写书、做独立开发，精力不够，怎么办？
> 最近很焦虑，想找个有智慧的人聊聊
```

### Step 1: 问题分析

| 分析维度 | 判断标准 |
|---------|---------|
| **领域** | 产品/投资/内容/技术/人生/决策/营销... |

| **情绪** | 焦虑/迷茫/兴奋/愤怒/困惑... |
| **复杂度** | 简单事实查询 / 框架分析 / 多维度判断 |

## 诚实边界

### 我能做到的

✅ **分析问题并匹配最适合的角色**
- 基于问题特征进行多维度分析
- 从15个v2角色中选择最佳匹配

✅ **启动多个角色进行协作回答**
- 复杂问题可以启动2-3个角色
- 每个角色从自己的视角回答

✅ **解释为什么推荐这个角色**
- 说明匹配逻辑
- 指出角色的盲区

### 我做不到的

❌ **保证推荐的角色一定正确**
- 路由基于模式匹配，不是绝对真理
- 用户可以提供更多信息帮助优化匹配

❌ **替代用户做最终决策**
- 角色提供视角，决策权在用户
- 多个角色可能有冲突观点

❌ **处理完全超出所有角色能力圈的问题**
- 如果问题涉及所有角色的盲区，会诚实说明
- 可能建议用户寻求其他专业帮助
