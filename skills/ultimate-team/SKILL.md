# 终极团队协作系统

## 核心理念

- 不只是找一个专家，而是组建一个完整的团队
- 团队成员互补，覆盖问题的所有维度
- 模拟真实企业中的"决策委员会"或"顾问团"

## 触发词

「终极团队」「组个队」「找个团队」「全面分析」「项目组」「帮我组建团队」「让团队看看」

## 团队组建流程（5步）

### Step 1: 需求分析

当用户触发本 Skill 时，首先对用户问题进行三维分析：

1. **问题类型判断**（选择最匹配的一项）：
   - 企业战略 / 产品开发 / 业务增长 / 组织管理 / 投资决策 / 危机处理 / 创业起步 / 品牌营销 / 技术战略 / 个人成长

2. **涉及维度分析**（可多选）：
   - 技术 / 市场 / 组织 / 财务 / 法律 / 心理 / 运营 / 品牌

3. **紧急程度与复杂度**：
   - 紧急程度：高（需快速决策）/ 中 / 低（可深度思考）
   - 复杂度：高（多维度交叉）/ 中 / 低（单一维度）

### Step 2: 团队组建

根据问题类型，从52位专家中选择 **4-6位** 组成团队。团队角色分工：

| 角色 | 人数 | 职责 |
|------|------|------|
| **队长** | 1位 | 最核心的专家，主导分析框架和最终判断 |
| **战略顾问** | 1-2位 | 提供战略层面、宏观视角的分析 |
| **执行顾问** | 1-2位 | 提供实操层面、落地方案的建议 |
| **挑战者** | 1位 | 专门提出质疑、反面观点和风险提示 |

**组建原则**：
- 优先使用下方「场景→团队映射表」中的预设组合
- 如果问题跨越多个场景，可从不同场景中各取专家混合组队
- 挑战者角色必须选择与团队主流观点不同的专家，确保有真正的对立面
- 有独立 Skill 的专家（v2版本），直接调用其 Skill 进行深度分析
- 无独立 Skill 的实战型专家（26位新增），基于公开信息中的核心思想框架进行分析

### Step 3: 逐位分析

每位专家从自己的核心框架出发，对问题进行分析：

- **队长**：建立分析框架，给出核心判断
- **战略顾问**：补充战略视角，指出关键变量
- **执行顾问**：给出具体可执行的行动建议
- **挑战者**：指出盲点、风险和反面论据

### Step 4: 团队讨论

模拟专家之间的真实对话：
- 专家之间互相回应、补充、反驳
- 形成交锋和共识
- 标注哪些是共识、哪些是分歧

### Step 5: 综合输出

整合所有专家观点，形成最终建议：
- 列出共识点和分歧点
- 给出综合建议和行动优先级
- 标注风险和注意事项

## 特殊规则

### 1. 跨场景组队
当用户问题涉及多个场景时：
- 从每个相关场景中选取最关键的1-2位专家
- 总人数控制在6位以内
- 挑战者必须来自与主流观点不同的阵营

### 2. 自定义组队
用户可以指定希望参与的专家，此时：
- 优先纳入用户指定的专家
- 自动补充缺失的角色（队长/战略/执行/挑战者）
- 如果用户指定的人选与问题场景不匹配，温和提醒并建议替代人选

### 3. 深度分析模式
当用户要求深度分析时：
- 对每位专家调用其独立 Skill（如有v2版本）
- 分析内容更加详细，包含具体方法论和案例
- 团队讨论环节更加深入，包含多轮交锋

### 4. 快速诊断模式
当用户需要快速回答时：
- 精简为3位专家（队长+执行+挑战者）
- 每位专家给出核心观点（3-5条要点）
- 直接给出综合建议，省略详细讨论过程

## 诚实边界

1. **模拟性质**：52位专家的分析基于公开信息（著作、演讲、访谈、案例）提炼而成，并非真人观点。所有分析都是对专家思想框架的模拟应用，不代表专家本人的实际判断。

2. **模式匹配局限**：团队组合基于场景模式匹配，虽然经过精心设计，但不一定是最优组合。实际商业决策需要结合具体情境和最新信息。

3. **观点冲突**：专家之间可能存在冲突观点（例如芒格的集中投资 vs 塔勒布的分散押注），这些冲突是刻意保留的。用户需要根据自身情况做出最终判断。

4. **时效性**：部分专家的思想框架可能随时间演进，本系统基于已知的公开信息构建，无法反映最新的观点变化。

5. **实战型专家**：26位新增的实战型企业专家中，部分没有独立 Skill，其分析基于公开报道和核心思想框架的提炼，深度可能不如有独立 Skill 的专家。
